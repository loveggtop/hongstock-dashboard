# 보유종목 통합분석 대시보드 업데이트

작업 폴더(이 스킬이 설치된 프로젝트 루트 = 종목매수가현황 폴더)의 데이터를 다시 읽어
`통합분석.html`을 재생성한다. 아래 순서를 그대로 따른다.

## 1. 거래내역 파싱

```bash
python3 .claude/skills/포트폴리오/scripts/parse_trades.py .
```

- 폴더의 모든 `KakaoTalk_Chat_*.csv`(NH투자증권·한국투자증권 체결 알림)를 파싱해
  `.claude/data/trades.json`을 갱신한다. 중복 알림은 자동 제거된다.
- 출력된 보유종목 요약을 확인하고, 파싱 실패(0건 등)나 새 증권사 형식이 보이면
  스크립트의 정규식을 새 메시지 형식에 맞게 확장한 뒤 다시 실행한다.

## 2. 새 분석 이미지 판독 (비전)

`.claude/data/image_analysis.json`의 `images` 키 목록과 폴더의 이미지 파일
(`*.jpeg`, `*.jpg`, `*.png`)을 비교해 **아직 판독하지 않은 이미지만** Read 도구로 열어
판독한다. 이미 판독된 이미지는 다시 읽지 않는다 (비용 절약).

각 이미지에서 추출해 `images`에 추가할 필드 (기존 항목과 동일한 스키마 유지):

```json
"파일명.jpeg": {
  "stock": "종목명", "code": "종목코드", "price": 현재가(숫자), "change_pct": 등락률,
  "peak_ref": "고점가격 (날짜) 대비 -N.NN%",
  "support": [1차지지선, 2차지지선], "resistance": [1차저항선, 2차저항선],
  "type": "차트요약" 또는 "매매전략",
  "comment": "차트요약일 때 핵심 코멘트",
  "strategy": { "buy": "...", "sell": "...", "stop": "..." }  ← 매매전략일 때
}
```

판독 후 `latest_by_stock`을 갱신한다. 최신 판별 기준:
- 같은 고점(peak_ref) 기준이면 **하락률(절대값)이 큰 쪽이 더 최신** 스크린샷이다.
- 고점 기준 날짜가 다르면 더 최근 고점 날짜를 참조하는 쪽을 우선하되,
  가격 흐름이 자연스럽게 이어지는지 교차 확인한다.

## 3. 대시보드 재생성

```bash
python3 .claude/skills/포트폴리오/scripts/build_dashboard.py .
```

`통합분석.html`이 갱신된다. 출력 요약(보유종목 수, 평가손익, 실현손익)을 확인한다.
표시 항목을 바꾸고 싶다는 요청이 있으면 이 스크립트를 수정한 뒤 재실행한다.

## 4. 확인 및 보고

```bash
open 통합분석.html
```

사용자에게 보고할 것:
- 이번에 새로 반영된 체결 건수·이미지 파일명
- 보유종목 수, 총 평가손익, 실현손익 누계
- 분석 이미지가 없는 보유 종목 (캡처 추가 안내)

## 5. GitHub Pages 배포 (설정된 경우)

폴더가 git 저장소이고 `docs/` 폴더가 있으면 GitHub Pages 배포용이다:

```bash
cp 통합분석.html docs/index.html
git add docs/index.html && git commit -m "통합분석 대시보드 갱신" && git push
```

git 저장소가 아니면 이 단계는 건너뛴다 (로컬 HTML만 갱신).

## 주의사항

- 이 대시보드는 개인 금융정보(보유종목·수량·손익)를 담고 있다. 외부 공개 채널에
  올리기 전에는 반드시 사용자에게 확인한다.
- 매매 판단은 사용자 몫이다. AI 분석 요약은 NH 차트분석 AI 캡처 내용을 옮긴 것이며,
  스킬이 자체적인 매수/매도 권유를 지어내지 않는다.
- 현재가는 캡처 시점 기준임을 대시보드와 보고에 항상 명시한다.
